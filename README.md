## Проект: Информационный сайт хакатонов России «Мои хакатоны»
Вертикаль: EdTech / HR-tech / IT-community

### Решаемая проблема: 
Отсутствие единой структурированной базы хакатонов в России. Информация о мероприятиях разрознена по сотням Telegram-каналов и сайтов, что затрудняет поиск возможностей для студентов и начинающих разработчиков. Проблема для организаторов - низкая вовлеченность аудитории из-за отсутствия прозрачного календаря событий.
Команда: Системный и бизнес-аналитик, команда разработки (Frontend/Backend), специалисты по машинному обучению

### Что сделано мной:
- Сбор и формализация требований (ТЗ) [2]
- Проектирование информационной архитектуры [4] [5]
- Разработка «Контракта данных»: спроектирована структура JSON-объекта хакатона [6]
- Разработаны функциональные требования к визуализации хакатонов по регионам России с возможностью быстрой фильтрации
- Сформирована дорожная карта проекта (Roadmap) [3]

### Результат:
Запущена платформа с высокой скоростью загрузки (до 3 сек) и поддержкой до 1000 одновременных пользователей. Автоматизирован процесс наполнения базы данных за счет парсинга, что позволило создать актуальный календарь мероприятий и повысить вовлеченность аудитории.
### Стек:
BPMN 2.0, UML (Use Case, Class, Object), CJM, User Story Mapping, Job Stories, ТЗ, SRS, BRD, API Design, JSON, SQL , LLM/NLP парсинг, системные интеграции (Telegram API), ПМИ, ПСИ, Miro, Figma, Draw.io.
